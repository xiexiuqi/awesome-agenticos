(AI 总结)
**1. 论文背景与目标**

- **AI 代理的兴起：** 代理 AI 系统（Agentic AI systems）在执行专业和个人任务方面日益强大，且只需有限的人工干预。研究表明，2025 年 AI 代理领域的研究和企业应用兴趣激增，论文数量和企业实验数量都显著增加（图 1）。
- **挑战：** 代理 AI 生态系统复杂、快速发展且文档记录不一致，给研究人员和政策制定者带来了追踪困难。
- **论文目标：** 针对这些挑战，本文介绍了“2025 AI Agent 指数”，旨在记录 30 个最先进的 AI 代理的起源、设计、能力、生态系统和安全特性，基于公开信息和与开发者的邮件往来。

**2. 指数构建方法**

- **数据来源：** 公开可用的信息（文档、网站、演示、已发表论文、治理文件）和与开发者的邮件通信。
    
- **筛选标准（三项全满足）：**
    
    - **代理性 (Agency)：** 必须满足所有四项标准：
        1. **自主性 (Autonomy)：** 至少达到中等自主水平（L2），即 AI 系统能独立完成大部分任务，但在关键决策上仍依赖用户输入。
        2. **目标复杂性 (Goal complexity)：** 能够通过长期规划、将复杂目标分解为子目标、做出时间依赖性决策来追求高层次目标，实际操作中至少能可靠地进行三次自主工具调用和高层次任务指定，无需分步指令。
        3. **环境交互 (Environmental interaction)：** 能够通过工具和 API 直接与世界交互，对环境产生实质性改变，需要对计算机有写入权限并能选择工具。
        4. **通用性 (Generality)：** 能够处理未明确的指令并适应新任务，在相关任务中展现多功能性，而非单一狭窄功能。
    - **影响力 (Impact)（满足一项即可）：**
        1. **公众关注度 (Public interest)：** 至少 10,000 次搜索量或 20,000 个 GitHub 星标。
        2. **市场重要性 (Market significance)：** 开发者市值或估值 ≥ 10 亿美元。
        3. **开发者重要性 (Developer significance)：** 开发者是 2024 年基础模型透明度指数、前沿模型论坛成员或相关安全承诺的签署方。
    - **实用性 (Practicality)（三项全满足）：**
        1. **公共可用性 (Public availability)：** 必须是公开可用的产品，不包括公司内部产品或有限预发布。
        2. **可部署性 (Deployability)：** 能够开箱即用，只需最少配置，无需软件工程专业知识，区别于开发框架。
        3. **通用目的 (General purpose)：** 能够实际执行通用任务，不限于特定领域。
- **代理分类：** 将代理系统分为三种主要交互类型：
    
    1. **带代理工具的聊天应用 (Chat applications with agentic tools)（12 个系统）：** 主要包含具有广泛工具访问权限的聊天界面，如通用编程代理。
    2. **基于浏览器的代理 (Browser-based agents)（5 个系统）：** 主要界面是浏览器或计算机使用，具有广泛的浏览器/计算机交互工具。
    3. **企业工作流代理 (Enterprise workflow agents)（13 个系统）：** 旨在可靠自动化业务任务的业务管理平台，通常实现为工作流构建器。

**3. 主要发现**

- **部署激增：** 大多数代理在 2024-2025 年发布或获得重大代理功能更新，表明代理部署近期激增，但底层模型（如 GPT-4）通常较旧，产品化（代理支架）正在加速。
- **透明度不足：**
    - **安全信息缺失：** 超过一半的安全相关字段（135/240）没有公开信息，企业代理和浏览器代理的缺失字段最多（图 6）。
    - **评估缺乏：** 很少有开发者发布代理特定的评估。大多数代理没有内部安全评估结果或第三方测试信息。只有少数前沿 AI 公司（如 OpenAI、Anthropic、Google）提供更全面的安全报告。
    - **AI 身份不披露：** 多数代理默认不向终端用户或第三方披露其 AI 性质。
    - **技术识别多样：** 许多代理融入正常网络流量，只有少数发布稳定的 User-Agent 字符串和 IP 地址范围用于验证。OpenAI ChatGPT Agent 独有地提供加密请求签名。
    - **Web 行为不规范：** 许多代理的 Web 行为（如 robots.txt 遵守、CAPTCHA 处理）未被记录，或被设计为绕过反机器人系统。
- **自主性与控制：**
    - **聊天助手：** 通常保持较低的自主性（L1-L3），通过回合制交互进行。
    - **浏览器代理：** 具有较高的自主性（L4-L5），但中途干预机会有限。
    - **企业平台：** 设计阶段自主性较低（L1-L2），部署后自主性较高（L3-L5），常由事件触发，无需人工干预。
    - **用户批准机制：** 敏感操作（如文件编辑、支付）需要明确批准，部分代理提供“监视模式”进行实时监督。
    - **执行追踪：** 常见，但透明度范围差异大。许多企业代理缺乏个体执行监控信息。
    - **停止机制：** 大多数代理允许用户启动停止，但部分缺乏细粒度控制。
- **生态系统交互：**
    - **基础模型集中：** 几乎所有索引中的系统都依赖少数闭源前沿模型家族（GPT、Claude、Gemini），美国和中国的模型开发者运行自己的专有模型。这种集中性带来了平台权力集中的风险。
    - **互操作性标准：** Model Context Protocol ([[MCP]]) 是主导的互操作性标准（20/30 个代理支持），Agent-to-Agent (A2A) 协议也得到部分企业代理支持。
- **风险评估挑战：** 代理生态系统的分层架构（基础模型、支架和编排层）使得评估实际的代理风险变得困难，因为风险不仅来自模型能力，还来自规划、工具、记忆和策略。

**4. 案例研究（突出不同类别代理的特点）**

- **ChatGPT Agent (代理聊天界面)：** 具有 L2-L4 自主性，对敏感操作要求用户批准。在托管虚拟计算机和沙盒终端中运行，网络访问受限。提供代理特定系统卡，并进行全面的安全评估（使用政策、越狱、幻觉、公平性、网络能力等）。独家采用 HTTP 请求加密签名。
- **Perplexity Comet (基于浏览器的代理)：** 具有 L4-L5 自主性，一旦启动便自主执行，无中途干预选项。缺乏代理特定的安全评估、第三方测试或基准性能披露。强调规避反机器人系统。
- **HubSpot Breeze Agents (企业代理构建器)：** 设计阶段 L1-L3，部署后 L5 自主性。用户通过模板配置工作流，默认要求批准操作。安全披露侧重合规性而非代理特定评估，将安全责任部分下放给用户。

**5. 结论与展望**

- **透明度不足的持续性：** AI 代理生态系统的信息记录仍然困难，缺乏结构化报告要求和大规模行业协作。
- **政策制定者：** 现有透明度期望大部分未实现，需要结构化要求。
- **开发者：** 识别出具体的信息披露空白（如代理特定系统卡、沙盒文档、Web 行为政策），改进披露可以区分负责任的开发者。
- **研究人员：** 本指数提供了研究代理透明度、生态系统集中度和责任碎片化的实证基线，强调需要针对代理行为而非仅模型能力进行评估的框架。
- **未来工作：** 扩展覆盖范围至内部和领域特定代理，更严格地审计和比较主要代理开发者的技术实践、报告和风险管理，并追踪这些模式随治理框架演进的情况。

---

总的来说，这篇论文通过“2025 AI Agent 指数”详细分析了当前代理 AI 系统的技术特性、自主性水平、安全实践和生态系统互动，揭示了在透明度和安全治理方面存在的显著不足，并为未来的研究、政策制定和开发提供了重要的洞察。
